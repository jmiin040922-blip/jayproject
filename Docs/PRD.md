# 마이링크 (MyLink) PRD (제품 요구사항 정의서)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 방문자들이 흩어져 있는 크리에이터의 온라인 흔적(포트폴리오, 블로그, SNS 등)에 한 번에 접근할 수 있도록 돕는 통합 멀티 링크 허브(Hub) 웹 서비스 제공
- **대상 사용자**: 
  - 여러 채널(유튜브, 인스타그램, 틱톡 등)에서 활동하는 **크리에이터**
- **기술 스택 (예정)**: Next.js (프론트엔드 프레임워크), Tailwind CSS & shadcn/ui (디자인 및 UI 컴포넌트), Firebase (OAuth 인증 및 데이터베이스)

---

## 2. 핵심 기능 목록

### ✅ 필수 기능 (MVP)
1. **Google 소셜 로그인 (Firebase Auth)**
2. **프로필 정보 관리 (이미지 URL, 표시 이름, 소개글)**
3. **링크 관리 CRUD (링크 생성, 조회, 수정, 삭제)**
4. **퍼블릭 프로필 페이지 제공 (방문자 열람용)**

---

## 3. 각 기능의 상세 설명

### 3.1 회원가입 및 로그인 (인증)
- **설명**: 사용자가 대시보드(관리자 페이지)에 접근하여 자신의 링크를 구성하기 위한 인증 기능
- **상세 요구사항**:
  - 비밀번호 관리 및 이메일 검증의 복잡도를 줄이기 위해 Firebase 기반 **단일 구글(Google) 소셜 로그인**만 지원.
  - 로그인 성공 시 사용자 전용 대시보드로 리다이렉트 처리.
  - 로그아웃 기능 제공.

### 3.2 프로필 관리
- **설명**: 방문자에게 보여질 사용자의 기본 정보(이름, 소개 등)를 설정하는 기능
- **상세 요구사항**:
  - **이미지 업로드 제외**: 사용자가 직접 이미지를 업로드하는 기능은 제공하지 않으며, 구글 로그인 시 연동된 프로필 사진 정보를 기본으로 활용합니다.
  - **프로필 정보 변경 (이름, 닉네임, 소개글)**: 프로필 화면에 노출될 **실제 이름(Username)**과, 고유 URL 슬러그(Slug)로 쓰일 **닉네임(displayName)**, 그리고 소개글(Bio)은 대시보드에서 자유롭게 **수정 가능**해야 합니다.
  - **초기 닉네임 및 URL 슬러그 연동**: 구글 회원가입 시 연동된 **지메일(Gmail) 계정 주소의 앞부분(@ 기호 이전 아이디)**을 파싱하여 `displayName`의 초기값으로 자동 설정합니다. 이 `displayName` 값이 별도 핸들 없이 사용자의 **고유 접속 URL의 슬러그(Slug)**로 동작합니다. (`mylink.com/displayName`)

### 3.3 링크 빌더 및 관리 (대시보드)
- **설명**: 퍼블릭 프로필에 표시될 각 링크 버튼들을 제어하는 대시보드 핵심 기능
- **상세 요구사항**:
  - **추가(Create)**: 링크에 표시될 '제목(Title)'과 도착 예정 'URL(주소)'을 입력하여 새로운 항목 생성.
  - **조회(Read)**: 사용자가 등록해 둔 전체 링크 리스트 확인. 각 카드의 형태로 제공됨.
  - **수정(Update)**: 기존 항목 수정 시 별도 페이지나 모달 대신 **인라인 편집(Inline Edit)** 방식을 적용하여 그 자리에서 즉각적으로 제목 및 URL 수정 처리를 수행.
  - **삭제(Delete)**: 더 이상 노출하고 싶지 않은 링크 삭제.
  - **파비콘 아이콘 설정 (Google API)**: 링크 아이템 생성 시 **구글 파비콘 API**(`https://s2.googleusercontent.com/s2/favicons?domain=...`)를 호출해 목적지 웹사이트의 파비콘을 가져와 자동 지정.
  - **제한 사항(추가)**: 대시보드 화면 내 실시간 모바일 뷰 미리보기(Mobile View Preview) 및 테마 커스터마이징 기능은 제공하지 않음.

### 3.4 퍼블릭 프로필 페이지 (공개 뷰)
- **설명**: 링크 관리자가 구성한 항목들이 대중에게 실제로 표출되는 랜딩 페이지
- **상세 요구사항**:
  - 인증을 거치지 않은 모든 외부 방문자가 접근 가능. (URL 구조: `mylink.com/displayName` 형태의 URL 슬러그 활용)
  - 상단 구역: 둥근 프로필 이미지와 함께 **실제 이름(Username)** 및 **닉네임(displayName)** 표출, 그 아래에 소개글(작게).
  - 하단 구역: 생성된 링크들이 세로형 블록/버튼으로 리스트업 됨. 각 링크 버튼에는 구글 API를 통해 불러온 파비콘이 아이콘으로 표시됨. 버튼 클릭 시 외부 URL로 새 창 열기(`_blank`).
  - 디자인 환경: 모바일 우선(Mobile-First) 디자인 구조로 구현하되, 테마(라이트/다크 등) 기능 없이 단일화된 깔끔한 디자인 톤으로 구성.

## 4. 데이터베이스 모델링 (Firebase Firestore)

### 4.1 루트 컬렉션 (`users`)
- **경로**: `users/{uid}`
- **설명**: 회원가입한 사용자의 고유 계정 및 프로필 데이터를 저장
- **주요 필드**:
  - `uid` (String): Firebase Auth가 제공하는 사용자의 고유 식별자
  - `username` (String): 프로필 방문자에게 보여질 **실제 사용자 이름** (구글 로그인 시의 이름으로 기본 연동)
  - `displayName` (String): 사용자 식별 및 **URL 슬러그(Slug)** 역할을 하는 **닉네임**. **구글 지메일 계정의 아이디 앞부분을 초기값**으로 자동 배정하며, 대시보드에서 언제든 수정 가능. (`mylink.com/displayName`)
  - `profileImage` (String): 프로필 이미지 URL (별도 업로드 없이 유지)
  - `bio` (String): 사용자 소개글

### 4.2 서브 컬렉션 (`links`)
- **경로**: `users/{uid}/links/{linkId}`
- **설명**: 각 사용자가 생성한 개별 링크 데이터를 해당 사용자 문서 하위의 서브 컬렉션으로 분리하여 관리
- **주요 필드**:
  - `id` (String): 링크 아이템의 고유 식별자
  - `title` (String): 링크 버튼에 표시될 제목
  - `url` (String): 이동할 목적지 링크 주소
  - `faviconUrl` (String): Google 파비콘 API로 자동 수집된 목적지 사이트의 파비콘 주소
  - `createdAt` (Timestamp): 생성 일자 (추후 조회/정렬에 활용)
