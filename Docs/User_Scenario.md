# 마이링크 (MyLink) 사용자 시나리오

서비스의 모든 기능적 흐름을 정의한 소유자(Owner)와 방문자(Visitor) 관점의 사용자 시나리오입니다. 
시나리오는 **"사용자는 [목적/이유]하기 위해 [행동/액션]한다"** 형식으로 기재되었습니다.

---

## 1. 👤 소유자 (Owner) 시나리오

### 가입 및 로그인 (Authentication)
1. **사용자는** 마이링크 서비스를 이용하기 위해 **구글(Google) 계정을 연동하여 가입 및 로그인을 진행한다.**
2. **사용자는** 서비스 이용을 종료하고 안전하게 빠져나가기 위해 **대시보드 또는 설정 메뉴에서 로그아웃 버튼을 클릭한다.**

### 프로필 관리 (Profile Management)
3. **사용자는** 방문자에게 실질적인 본인의 정체성을 드러내기 위해 **대시보드에서 프로필에 노출될 실제 이름(Username)을 수정한다.**
4. **사용자는** 자신만의 고유한 접속 주소를 만들기 위해 **초기 배정된 지메일 아이디 대신 원하는 단어로 닉네임(displayName)을 수정한다.**
5. **사용자는** 방문자들에게 자신을 보다 구체적으로 소개하기 위해 **대시보드에서 자기소개글(Bio) 텍스트를 기입하고 수정한다.**

### 링크 관리 (Link Management: CRUD)
6. **사용자는** 자신의 포트폴리오나 외부 채널을 홍보하기 위해 **대시보드에서 '제목(Title)'과 'URL 주소'를 입력하여 새로운 링크 버튼을 추가한다.**
7. **사용자는** 방금 추가한 링크의 성격을 직관적으로 파악할 수 있도록 도우기 위해 **구글 API를 통해 화면에 자동 추출되어 나타난 목적지 사이트의 파비콘(Favicon)을 확인한다.**
8. **사용자는** 오타가 난 링크 제목이나 변경된 URL을 빠르게 정정하기 위해 **링크 목록 내 해당 항목을 클릭하여 즉각적으로 내용을 수정(인라인 편집)한다.**
9. **사용자는** 기한이 만료되었거나 더 이상 홍보할 필요가 없는 링크를 없애기 위해 **해당 링크 블록 우측의 '삭제' 액션 버튼을 클릭한다.**
10. **사용자는** 생성한 여러 링크들이 방문자 화면에서 어떤 순서로 보이는지 파악하기 위해 **가장 최근에 추가한 링크가 최상단에 자동 배치되는 것(최신순 정렬)을 확인한다.**

### 서비스 공유 및 홍보 (Sharing)
11. **사용자는** 완성된 자신의 통합 프로필을 인스타그램이나 틱톡과 같은 SNS 프로필 영역에 홍보하기 위해 **대시보드 상단에 위치한 '내 링크 복사' 버튼을 클릭한다.**

---

## 2. 👀 방문자 (Visitor) 시나리오

### 페이지 접근 경험 (Navigation)
12. **사용자(방문자)는** 크리에이터의 전체 채널 정보를 한 번에 열람하기 위해 **크리에이터가 타 SNS에 홍보한 마이링크 주소(`mylink.com/닉네임`)를 클릭하여 접속한다.**
13. **사용자(방문자)는** 주소를 잘못 입력했거나 이미 삭제된 계정으로 접근했을 경우 자신이 잘못된 경로에 있음을 명확히 안내받기 위해 **"존재하지 않는 페이지입니다"라는 404(Not Found) 에러 화면을 확인한다.**
14. **사용자(방문자)는** 현재 접속한 페이지가 자신이 찾던 크리에이터의 프로필이 맞는지 검증하기 위해 **화면 상단에 노출된 둥근 프로필 사진과 실제 이름(Username), 닉네임(displayName), 그리고 간단한 소개글(Bio)을 확인한다.**

### 소통 및 상호작용 (Interaction & Analytics)
15. **사용자(방문자)는** 나열된 여러 목적지 중 본인의 관심사에 맞는 외부 사이트로 이동하기 위해 **원하는 내용이 적혀있고 파비콘이 표시된 개별 링크 버튼을 클릭한다.**
16. **사용자(방문자)는** 크리에이터의 콘텐츠를 탐색하면서도 기존 마이링크 허브(Hub) 창을 잃어버리지 않기 위해 **클릭한 링크의 외부 사이트가 기존 탭이 아닌 새 창(`_blank`)으로 열리는 것을 확인한다.**
17. **(시스템 백그라운드) 시스템은** 어떤 콘텐츠가 방문자에게 매력적인지 파악하여 전체 조회수(PV) 없이 오직 '클릭수'만 측정하기 위해, **방문자가 특정 링크 버튼을 클릭하는 즉시 데이터베이스 상에 해당 링크의 '클릭 수(Click Count)'를 1회 상승시킨다.**
